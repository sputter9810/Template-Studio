# Design System

## Brand Personality
Reassuring, grounded, professional, calm.

## Colour System
### Primary
Deep sage or slate

### Secondary
Muted eucalyptus

### Accent
Warm sand

### Neutrals
White, soft grey, stone

### Feedback / Utility Colours
Standard accessible utility palette

## Typography
### Font Pairing / Style Direction
Readable sans-serif with softly elevated heading style.

### Type Scale
- H1: clamp(2.6rem, 6vw, 4.8rem), strong visual priority
- H2: clamp(2rem, 4vw, 3.2rem)
- H3: 1.25rem–1.6rem
- Body: 1rem–1.125rem with generous line-height
- Small: 0.875rem–0.95rem

## Spacing System
Use an 8px-based system with generous section padding. Major sections should breathe on desktop and remain compact but readable on mobile.

## Radius / Shadows / Borders
Use medium corner radius, subtle border lines, and restrained shadowing. Let the niche mood decide whether the finish feels crisp, soft, premium, or energetic.

## Layout System
Use a centred container, simple responsive grid, and repeatable split layouts. Default to one-column mobile, then 2–3 column layouts as space allows.

## Breakpoints
- Mobile: 0–759px
- Tablet: 760–1023px
- Desktop: 1024–1439px
- Large Desktop: 1440px+

## Components
### Buttons
Primary CTA should stand out clearly. Secondary CTA should support comparison or exploration without competing with the main action.

### Cards
Use shared `.card` styling with niche-appropriate variants for services, menu items, pricing, testimonials, or featured offers.

### Forms
Inputs should feel clean, clearly labelled, and low-friction. Keep booking/enquiry forms short.

### Navigation
Sticky header is preferred. Mobile navigation should remain lightweight and easy to scan.

### Badges / Tags
Use only where they improve scanning, such as bestseller, beginner-friendly, featured package, or available today.

## Imagery / Placeholder Rules
Use realistic placeholders that match the niche mood and product. Do not rely on generic gradients alone when product or venue photography would better communicate the offer.

## Interaction Style
Keep hover and focus states obvious, smooth, and lightweight. Avoid decorative animation that slows the page or distracts from CTA paths.

## Accessibility Rules
- Maintain strong text contrast and readable body sizes
- Make focus states clearly visible
- Ensure buttons, nav items, and forms are keyboard-usable
- Do not hide core information behind hover-only states

## Notes
Less luxury than spa, more clarity and trust. Cleaner sections, stronger forms, and practical content blocks.
