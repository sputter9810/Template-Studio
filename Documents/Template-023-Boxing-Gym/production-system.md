# Website Production System

## Purpose
This production system is used to turn a reusable static website template framework into sellable niche-ready products. Each template should feel specific to the buyer, but still be built from a repeatable shared system.

## Business Model / Product Type
Static website templates for small businesses, hospitality brands, studios, clinics, and service businesses. Templates are sold as reusable digital products and can also be adapted for client work.

## Standard Production Stages
1. Define the niche, buyer, and conversion goal
2. Fill out the project brief and content guide
3. Select suitable components and sections from the component library
4. Define the design system and visual direction
5. Scaffold the HTML/CSS structure
6. Write realistic placeholder content and CTA paths
7. Apply responsiveness, accessibility, and interaction rules
8. QA the template against the standard and checklist
9. Prepare product documentation and previews
10. Package the template for sale or reuse

## Folder Structure Rules
Every template should follow the standard static structure:

```text
src/
  index.html
  assets/
    css/
      base/
      layout/
      components/
      sections/
    js/
    images/
docs/
preview/
```

## Coding Standards
- Use semantic HTML with anchor-linked one-page navigation where appropriate
- Reuse shared class names and component structures before creating anything new
- Keep JavaScript lightweight and only for navigation or simple interactions
- Keep code readable, modular, and easy for buyers to customise

## Design Standards
- Prioritise clarity, conversion, and niche relevance
- Use a consistent spacing and typography system
- Keep visual hierarchy obvious on mobile first
- Use imagery and placeholder content that support the niche without locking the buyer in

## Output Requirements
- One clean index page scaffolded from the library
- Documentation that explains how to customise niche content
- Reusable placeholder content that feels realistic
- Packaging that is ready for storefront upload or client handoff

## Agent / AI Behaviour Rules
- Prefer existing library patterns before inventing new structures
- Keep copy conversion-focused and niche-specific
- Report what was reused, added, or customised
- When uncertain, choose the simplest reusable solution that still fits the niche

## Packaging Rules
Final delivery should include the source files, documentation, preview assets, and only the necessary supporting files. The pack should feel productised rather than like a raw dev folder.

## Notes
Use the same production language across all niches so each template remains part of a single coherent product line.

## Scaling Rules
- Each base template must support 3+ niche variants
- Variants should reuse 70–80% of structure
- Only content, imagery, and section emphasis should change

## Output Targets
- Minimum 3 templates per week
- Reuse before creating new components
