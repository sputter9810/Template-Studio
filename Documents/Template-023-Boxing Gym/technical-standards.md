# Technical Standards

## Approved Stack
Static HTML, CSS, and lightweight JavaScript.

## Disallowed Tools / Dependencies
- Heavy frameworks unless a later product line explicitly requires them
- Overbuilt animation libraries in core packs

## File Naming Rules
Use lowercase kebab-case for files and folders. Keep names descriptive and reusable.

## Folder Naming Rules
Organise CSS by base, layout, components, and sections. Keep assets grouped logically by type.

## HTML Standards
- Use semantic landmarks and logical heading hierarchy
- All forms must have labels and sensible input types
- Keep anchor IDs clear and predictable

## CSS Standards
- Follow a layered architecture with variables, typography, layout, components, and sections
- Prefer reusable classes over niche-specific one-off selectors
- Keep selectors shallow and maintainable
- Build mobile-first and scale upward

## JavaScript Standards
- Use JS only for small enhancements such as nav toggles
- Avoid complex state or app-like behaviour in brochure templates
- Keep interactions accessible and progressive
- Keep payloads small

## Asset Rules
Store images, icons, and media in predictable folders. Use placeholders that buyers can replace easily.

## Responsiveness Requirements
- Every section must remain readable on small screens
- Card grids must collapse cleanly
- Navigation and CTA actions must remain usable on touch devices

## Accessibility Requirements
- Respect contrast and readable type size
- Preserve keyboard access to nav, forms, and buttons
- Use meaningful labels, alt text, and focus styles

## Performance Expectations
- Avoid unnecessary JS
- Keep image strategy simple and replaceable
- Ship only files needed for the product

## SEO / Metadata Rules
- Include a page title and meta description placeholder
- Use a single H1 per page
- Structure content for scanability and local-business discoverability where relevant

## Documentation Requirements
- Include a buyer-facing README
- Include setup/customisation notes
- Include licensing information if required by storefront

## Notes
These standards are intended to keep templates easy to sell, easy to edit, and easy to extend.
