# QA Checklist

## Structure
- [ ] All required files exist
- [ ] Folder structure is correct
- [ ] File names follow conventions
- [ ] No unnecessary files are included

## HTML
- [ ] Semantic HTML is used
- [ ] Heading hierarchy is logical
- [ ] Links are valid
- [ ] Forms have labels
- [ ] Images have alt text where needed

## CSS / Design
- [ ] Spacing is consistent
- [ ] Typography is consistent
- [ ] Colours follow the design system
- [ ] Components are visually consistent
- [ ] No layout breakage is visible

## Responsive Behaviour
- [ ] Mobile layout works
- [ ] Tablet layout works
- [ ] Desktop layout works
- [ ] Navigation works across screen sizes
- [ ] No overflow / clipping issues

## Accessibility
- [ ] Colour contrast is acceptable
- [ ] Focus states are visible
- [ ] Interactive elements are usable by keyboard
- [ ] Form inputs are clearly identified

## Performance / Technical
- [ ] No broken asset paths
- [ ] CSS and JS are not unnecessarily bloated
- [ ] Images are handled appropriately
- [ ] Metadata is included where required

## Product / Packaging
- [ ] README exists
- [ ] LICENSE exists if required
- [ ] Preview/demo assets are included if required
- [ ] Template is ready for packaging or upload

## Final Review
- [ ] The output matches the project brief
- [ ] The output matches the production system
- [ ] The output is polished enough to hand off or sell

[Record any issues, compromises, or next steps.]
